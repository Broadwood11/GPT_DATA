1 · What IGEL Actually Is

The hardened endpoint OS that makes ControlUp data better

Foundation layer · No drift · No local data

IGEL is a modern, hardened endpoint OS designed for secure digital workspaces. It delivers a Preventative Security Model™ where:

  • • No local data is ever stored on the endpoint.
  • • Only trusted, cryptographically signed apps are allowed to execute.
  • • Devices behave as immutable endpoints, locked down, policy-driven, centrally managed.

IGEL OS is controlled via the Universal Management Suite (UMS) and extended through the IGEL App Portal, where ControlUp components are pre-validated and ready to deploy.

For ControlUp, think of IGEL as:

  • • The standardized device baseline your telemetry relies on.
  • • A way to eliminate rogue apps and inconsistent OS states.
  • • A security layer that makes Zero Trust and DEX stories land.

When IGEL locks the edge, ControlUp can focus on what it does best: real-time visibility, diagnostics, and automation.

8 · Joint Value Architecture

Zero Trust from cloud to endpoint

Technical + operational alignment

Technical alignment

IGEL and ControlUp form a continuous, intelligent loop of standardize → monitor → diagnose → remediate.

  • • IGEL provides the immutable, policy-driven endpoint OS, centrally managed via UMS.
  • • ControlUp Edge DX and Remote DX stream real-time metrics (CPU, Wi-Fi, latency, UC quality, etc.).
  • • ControlUp ONE correlates endpoint, session, and network data across VDI/DaaS and SaaS.
  • • Telemetry flows outbound-only, supporting secure, internet-facing architectures and Zero Trust principles.

The result is true end-to-end visibility from hardened device to virtual session and back.

Operational alignment

  • • Zero-touch updates via IGEL App Portal and UMS packaging.
  • • Proven scalability from hundreds to tens of thousands of endpoints.
  • • Joint support model that reduces POC friction and speeds production deployment.
  • • Works across hybrid OS estates and branch/remote environments.

Practically, this removes silos between EUC, network, and endpoint teams and shifts IT from firefighting to proactive assurance.
