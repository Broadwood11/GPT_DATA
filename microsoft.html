2 · Joint Value Architecture

Zero Trust from Azure to the endpoint.

1. Architecture & Interoperability

  • IGEL endpoints connect directly to Windows desktops hosted in Azure (AVD & Windows 365) — no local Windows, no local workload.
  • Sessions run in the Microsoft Cloud; endpoints remain stateless, centrally governed, and read-only.
  • IGEL Cloud Gateway and secure reverse connect (HTTPS/RDP) enable remote, off-domain access without VPN sprawl.

2. Identity, Security & Management

  • Entra ID (Azure AD), Conditional Access, and MFA handle user identity; IGEL supports FIDO2, smartcards, and device compliance natively.
  • IGEL Universal Management Suite (UMS) complements Microsoft Intune for joint policy control, compliance reporting, and automation.
  • Co-engineered Teams media offload, USB redirection, smartcard handling, and GPU acceleration deliver a first-class user experience.

3. Operations, Lifecycle & Scale

  • Deploy at scale using conversion kits or IGEL UD Pocket to transform existing PCs into secure Azure access points.
  • Centralized, atomic patching through Intune + UMS; apps update in Azure, not on the endpoint.
  • Supports global scale across Azure regions and hybrid deployments via Azure Stack HCI while coexisting with Citrix or Omnissa during transitions.

4. Preventative Security Architecture

Instead of detecting breaches after the fact, IGEL + Microsoft remove the easy paths to compromise:

  • • No local Windows OS to exploit.
  • • No sensitive data stored on endpoints.
  • • Identity-driven access with Conditional Access and MFA.
  • • Immutable OS image = no configuration drift.

Use this framing with CISOs and compliance leaders: you’re not adding another agent — you’re changing the attack surface.

4 · Field Guide

IGEL for Microsoft Sellers: How to position it in the deal.

WHAT IGEL ACTUALLY IS

IGEL is a modern, hardened endpoint OS — a light, read-only layer that replaces Windows on the device and connects users to Windows desktops in Azure.

Think of it as the secure access layer that makes Windows 365 and AVD deployments faster to roll out, cheaper to run, and harder to breach.

30-second joint pitch:

“Microsoft delivers Windows from the cloud. IGEL ensures nothing risky happens at the edge. Together, we make hybrid work secure, sustainable, and simple to manage.”

SIGNALS TO BRING IGEL IN

  • • Customer mentions ransomware, endpoint risk, or compliance audits.
  • • “Can we reuse existing PCs for Cloud PCs·”
  • • Patching fatigue or high endpoint support tickets.
  • • Citrix/Omnissa → AVD or Windows 365 migration.
  • • Concerns about data leaving the data center or cloud.

Your move:

When you hear these, position IGEL as the way to remove endpoint objections so the cloud desktop deal closes faster.

HOW IGEL STRENGTHENS MICROSOFT

  • • Speeds deployment: boot-to-cloud in minutes using UD Pocket or conversions.
  • • Simplifies management: Intune + UMS reduce console sprawl and tickets.
  • • Reduces risk: no local Windows, no local data, strong FIDO2/MFA support.
  • • Boosts Teams UX: co-engineered media offload and performance tuning.
  • • Extends device life: run Windows 11 from Azure on 5-year-old PCs.

Net effect: the Microsoft workspace story becomes cleaner, more defensible, and easier to champion.
