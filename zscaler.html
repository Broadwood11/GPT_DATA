2 · Joint Value Architecture

How IGEL + Zscaler Fit Technically

Use this for whiteboarding and slides

Zero Trust from Cloud to Endpoint

At its core, the IGEL-Zscaler architecture is simple: Zscaler secures the path, brokering every connection, and IGEL secures the device, enforcing an immutable, dataless OS at the edge.

Endpoint layer · IGEL OS

  • - Immutable, read-only OS — no writable partitions
  • - No local data, minimal attack surface, no drift
  • - Centralized control via Universal Management Suite (UMS)
  • - Deploy as reimage or USB boot on existing x86 hardware

Access layer · Zscaler Zero Trust Exchange

  • - ZIA: secure internet & SaaS access
  • - ZPA: app-level Zero Trust access to internal apps
  • - ZCC app packaged in IGEL App Portal
  • - Cloud Browser Isolation & PRA for agentless/BYOD access

Security synergy

  • - IGEL locks down USB, local apps, and write access
  • - Zscaler applies inline inspection, isolation, and DLP
  • - Malware has nowhere to land; data has nowhere to hide

Operational view

  • - IGEL OS rollout via reimage, USB, or repurposed hardware
  • - Zscaler is SaaS-delivered — identity + policy, no boxes
  • - UMS + Zscaler console + ZDX = end-to-end visibility

Stack Mapping

1

Cloud / Network

Zscaler Zero Trust Exchange brokers all access to SaaS, internet, and private apps.

2

Endpoint / Edge

IGEL OS turns endpoints into immutable access devices with no local attack surface.

3

Identity / Policy

Customer’s IdP + Zscaler policy engine decide who gets to what; IGEL enforces how the device can behave.

Risk reduction snapshot

Endpoint attack surface down as much as 95%
VPN / VDI infra reliance down 40-60%

Stats are indicative, based on field deployments & customer case studies.
