What IGEL Actually Is

Foundation

Preventative security model

IGEL is a modern, hardened endpoint OS designed around the Preventative Security Model. It removes the local attack surface and turns any compatible x86 device into a secure, compliant Zero Trust endpoint.

Immutable by design

Read only, modular OS. Endpoints always boot into a known good state.

Dataless by intent

No user data, apps, or credentials stored locally. Nothing meaningful to steal or ransom.

Cloud first by nature

Optimized for SaaS, VDI, DaaS, and browser workspaces including Zscaler Cloud Browser Isolation.

Managed at scale

Universal Management Suite (UMS) provides one-to-many policy, configuration, and firmware control.

In short: IGEL turns any device into a secure, compliant, and easily managed Zero Trust endpoint that is ideal for Zscaler deployments.

IGEL is not a hardware vendor story. It is a software platform that replaces traditional desktop OS at the edge for users who primarily access resources through the network and browser.

Where IGEL Fits in the Zscaler Stack

Architecture

Origin + path

Zscaler governs what a user can access and how traffic flows. IGEL governs where that access originates, ensuring the endpoint is clean, compliant, and uncompromised.

Layer Zscaler IGEL
Network and app access Zscaler Zero Trust Exchange brokers and inspects all connections. Provides a trusted, tamper-resistant device to originate those connections.
Identity and session control ZPA/ZIA enforce least privilege and session validation. Ensures the session begins on an immutable, policy-locked OS.
Browser isolation Cloud Browser Isolation keeps web threats off the device. Prevents any residue or cached data from persisting locally.
Compliance DLP, audit trails, and policy enforcement. No regulated data at rest; consistent device posture for audits.
Together they deliver Zero Trust from cloud to endpoint, closing the last mile risk that Zscaler alone cannot see: unmanaged or untrusted devices.
