02

IGEL in plain English

What it is

A hardened endpoint OS

IGEL OS replaces the traditional PC stack with a read-only, centrally managed OS designed for secure access to virtual and cloud workspaces.

What it changes

No data at the edge

Reduce breach blast-radius by keeping apps/data in the datacenter or cloud — not on endpoints that drift, get lost, or get patched inconsistently.

How it’s managed

UMS / Cloud Gateway

One-to-many policies. Atomic updates. Fast recovery by reflash/reset. Manage globally at scale with a consistent “golden” posture.

Customer-friendly translation

IGEL turns any compatible device (PC / laptop / thin client) into a stateless, secure access terminal for Nutanix Frame or VDI — with less patching, less drift, and less risk.

Preventative Security Model™
Hover / tap

Design out the failure modes: no local data, reduced attack surface, enforced integrity.

Translation: fewer things to patch, fewer places to hide malware, faster to recover.
Trusted App Platform
Hover / tap

Only trusted/signed components run — helps keep endpoint posture consistent.

Use this line: “It reduces endpoint drift and prevents ‘random software’ from becoming your new attack surface.”
Atomic Updates
Hover / tap

Update the OS as a single unit → fewer broken states and easier rollbacks.

Customer benefit: less time chasing patch regressions across thousands of endpoints.

        07

        Objections & crisp answers

        IGEL doesn’t replace your delivery stack — it stabilizes the endpoint layer. You get fewer tickets from patch drift, faster rollout, better security posture, and easier recovery — while continuing Frame/Citrix/Horizon on AHV.
        It’s an enterprise endpoint OS that can run on many device types — PCs, laptops, and thin clients — with centralized policy and a preventative security model. It’s about posture and control, not a hardware category.
        The joint solution is validated with common collaboration tools and peripherals. The right next step is a quick pilot on your top devices + workflows (multi-monitor, USB, cameras) to confirm UX.
        These plays are strongest when users can reach the delivery layer (Frame/AHV/VDI). For fully offline environments, treat this as a fit check and align on constraints early.
        It’s less change where it hurts most: endpoints. IGEL can run on existing hardware, with fast recovery (reflash/reset), and centralized policy. Net effect: fewer moving parts across the fleet.
        Anchor on three levers: device reuse (capex avoidance), reduced support load (patch/drift tickets), and risk reduction (no data at rest). Start with a fast pilot and baseline current ticket + refresh costs.
